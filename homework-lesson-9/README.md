# Мультиагентна система на базі MCP + ACP (Homework 9)

Цей проект є реалізацією дев'ятого домашнього завдання, де класична мультиагентна система (Supervisor + Sub-agents) була переведена на сучасну мікросервісну архітектуру за допомогою протоколів **MCP** та **ACP**.

## 🏗️ Архітектура системи

Система розподілена на незалежні runtime-поверхні, що спілкуються виключно через мережеві протоколи:

1.  **Supervisor CLI (Local):** Головний оркестратор. Не містить логіки агентів або інструментів. Керує потоком `Plan -> Research -> Critique` через ACP та ініціює збереження звітів через MCP.
2.  **ACP Server (Port 8903):** "Місток" для агентів. Публікує ролі `planner`, `researcher` та `critic` як віддалені сервіси.
3.  **SearchMCP (Port 8901):** Сервіс інструментів пошуку (`web_search`, `read_url`, `knowledge_search`). Використовується всіма ACP агентами.
4.  **GitHubMCP (Port 8904):** Сервіс для роботи з репозиторіями (`github_list_directory`, `github_get_file_content`).
5.  **ReportMCP (Port 8902):** Спеціалізований сервіс для персистентності звітів.
6.  **Qdrant:** Векторна база даних для RAG.

## 🚀 Як запустити

### Спосіб 1: Docker Compose (Рекомендовано)
Піднімає весь стек мікросервісів однією командою:
```bash
docker compose up --build
```
Для взаємодії з CLI підключіться до контейнера:
```bash
docker attach homework-lesson-9-supervisor-cli
```

### Спосіб 2: Локальний запуск (Development)
1.  **Завантаження даних:** `npm run ingest`
2.  **Запуск усіх серверів та CLI:** `npm run dev:stack`
    *(Скрипт автоматично запустить 4 сервери у фоні та відкриє інтерфейс Supervisor)*
