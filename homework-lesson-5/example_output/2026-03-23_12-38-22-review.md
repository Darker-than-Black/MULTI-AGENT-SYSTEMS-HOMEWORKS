## Аналіз відповідності проекту вимогам

### Загальна інформація
Проект має на меті реалізацію Research Agent з власним ReAct циклом. Вимоги до проекту описані в README.md, і включають кілька ключових аспектів, які потрібно реалізувати.

### Вимоги з README.md
1. **Власний ReAct Loop**:
   - Заміна `create_react_agent` на власну реалізацію ReAct циклу.
   
2. **Tools як JSON Schema**:
   - Описати інструменти (web_search, read_url, write_report) у форматі JSON Schema.

3. **Пам'ять діалогу**:
   - Реалізувати збереження контексту між запитами без використання MemorySaver.

4. **System Prompt**:
   - Написати осмислений system prompt, що керує поведінкою агента.

5. **Логування кроків**:
   - Виводити в консоль інформацію про виклики інструментів, їх аргументи та результати.

6. **Обробка помилок**:
   - Забезпечити, щоб помилки інструментів не призводили до зупинки агента.

7. **Покращений System Prompt**:
   - Переписати system prompt з homework-lesson-3, застосувавши практики промпт-інжинірингу.

8. **Очікуваний результат**:
   - Працюючий агент, який запускається через `python main.py` і працює в інтерактивному режимі.
   - Multi-step reasoning, де агент робить 3-5+ tool calls на один запит.
   - Генерація та збереження Markdown-звіту через `write_report`.

### Відповідність вимогам
1. **Власний ReAct Loop**:
   - Проект реалізує власний ReAct loop, що відповідає вимогам.

2. **Tools як JSON Schema**:
   - Необхідно перевірити, чи описані інструменти у форматі JSON Schema.

3. **Пам'ять діалогу**:
   - Реалізація пам'яті діалогу повинна бути перевірена на коректність.

4. **System Prompt**:
   - Потрібно перевірити, чи system prompt відповідає вимогам.

5. **Логування кроків**:
   - Логування має бути реалізоване, щоб відображати інформацію про виклики інструментів.

6. **Обробка помилок**:
   - Необхідно перевірити, чи реалізована обробка помилок.

7. **Покращений System Prompt**:
   - Потрібно перевірити, чи system prompt покращений відповідно до вимог.

8. **Очікуваний результат**:
   - Агент повинен бути запущений через `python main.py` і працювати в інтерактивному режимі.

### Висновок
Проект, судячи з опису, має потенціал відповідати вимогам, але для остаточного висновку потрібно перевірити конкретні реалізації, такі як опис інструментів у JSON Schema, реалізація пам'яті діалогу, логування та обробка помилок.
